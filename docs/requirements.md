## 1. 项目背景

### 1.1 业务背景
中小型企业在经营过程中，普遍面临采购、库存、销售、财务等环节管理混乱、数据分散的问题。手工记账容易出错，库存不透明导致积压或缺货，业务流程缺乏规范化管控。企业需要一套轻量、实用的ERP系统，实现进销存与财务的一体化管理。

### 1.2 项目目标
构建一套面向中小型企业的轻量级ERP系统，聚焦**采购管理、库存管理、销售管理、财务管理、系统管理**五大核心模块，帮助企业规范业务流程、实时掌控经营数据、提升运营效率。

### 1.3 预期收益
- 规范业务流程，减少人为操作失误
- 实时掌握库存状态，降低库存积压
- 统一管理采购与销售数据，加速资金周转
- 自动化财务记录，提升对账效率
- 数据可视化，辅助经营决策

## 2. 用户角色分析

### 2.1 角色定义

| 角色 | 描述 | 主要使用模块 |
|------|------|------------|
| 系统管理员 | 负责系统配置、用户权限管理 | 系统管理 |
| 企业负责人 | 查看经营数据、审批关键业务 | 工作台、报表 |
| 采购员 | 日常采购操作、供应商管理 | 采购管理 |
| 仓库管理员 | 出入库操作、库存盘点 | 库存管理 |
| 销售员 | 客户管理、销售订单处理 | 销售管理 |
| 财务人员 | 收付款管理、费用记录、报表 | 财务管理 |

### 2.2 用户规模预估
- 单企业用户数：10-200人
- 日活用户比例：50%-70%
- 并发用户峰值：20-50人

## 3. 功能需求

### 3.1 系统管理模块

#### 3.1.1 用户管理
- **FR-SYS-001**: 支持用户的增删改查操作
- **FR-SYS-002**: 支持用户状态管理（启用/停用）
- **FR-SYS-003**: 支持管理员重置用户密码
- **FR-SYS-004**: 支持用户关联角色

#### 3.1.2 角色管理
- **FR-SYS-010**: 支持角色的增删改查操作
- **FR-SYS-011**: 支持为角色分配菜单权限
- **FR-SYS-012**: 支持RBAC权限控制模型

#### 3.1.3 部门管理
- **FR-SYS-020**: 支持树形组织结构管理
- **FR-SYS-021**: 支持部门的增删改查

#### 3.1.4 菜单管理
- **FR-SYS-030**: 支持菜单的增删改查（目录/菜单/按钮）
- **FR-SYS-031**: 支持菜单的树形展示与排序

#### 3.1.5 日志管理
- **FR-SYS-040**: 支持操作日志记录与查询
- **FR-SYS-041**: 支持登录日志记录与查询

#### 3.1.6 字典管理
- **FR-SYS-050**: 支持数据字典的维护（如状态、类型等枚举值）

### 3.2 采购管理模块

#### 3.2.1 供应商管理
- **FR-PUR-001**: 支持供应商信息登记（名称、联系人、电话、地址、银行信息）
- **FR-PUR-002**: 支持供应商状态管理（合作中/暂停合作）
- **FR-PUR-003**: 支持供应商列表查询与导出

#### 3.2.2 采购订单管理
- **FR-PUR-010**: 支持创建采购订单（选择供应商、添加采购明细）
- **FR-PUR-011**: 支持采购订单审核（提交→审核通过/驳回）
- **FR-PUR-012**: 支持采购订单状态跟踪（待审核→已审核→部分入库→已完成→已关闭）
- **FR-PUR-013**: 支持采购订单打印

#### 3.2.3 采购入库
- **FR-PUR-020**: 支持根据采购订单生成入库单
- **FR-PUR-021**: 支持部分入库（多次到货）
- **FR-PUR-022**: 入库后自动更新库存数量

#### 3.2.4 采购退货
- **FR-PUR-030**: 支持创建采购退货单
- **FR-PUR-031**: 退货后自动扣减库存

#### 3.2.5 采购统计
- **FR-PUR-040**: 支持按时间、供应商维度统计采购金额
- **FR-PUR-041**: 支持采购明细报表导出

### 3.3 库存管理模块

#### 3.3.1 商品/物料管理
- **FR-INV-001**: 支持商品信息管理（编码、名称、分类、规格、单位、单价）
- **FR-INV-002**: 支持商品分类管理（树形结构）
- **FR-INV-003**: 支持商品数据导入导出（Excel）

#### 3.3.2 仓库管理
- **FR-INV-010**: 支持仓库信息维护（名称、地址、负责人）
- **FR-INV-011**: 支持多仓库管理

#### 3.3.3 入库管理
- **FR-INV-020**: 支持采购入库（关联采购订单）
- **FR-INV-021**: 支持退货入库（关联销售退货）
- **FR-INV-022**: 支持其他入库（手动录入）
- **FR-INV-023**: 入库自动更新库存

#### 3.3.4 出库管理
- **FR-INV-030**: 支持销售出库（关联销售订单）
- **FR-INV-031**: 支持报损出库
- **FR-INV-032**: 支持其他出库（手动录入）
- **FR-INV-033**: 出库自动扣减库存

#### 3.3.5 库存盘点
- **FR-INV-040**: 支持创建盘点单
- **FR-INV-041**: 支持盘点差异自动计算
- **FR-INV-042**: 支持盘盈盘亏调整

#### 3.3.6 库存预警
- **FR-INV-050**: 支持设置安全库存量
- **FR-INV-051**: 库存低于安全库存时自动提醒

#### 3.3.7 库存查询
- **FR-INV-060**: 支持实时库存查询（按商品、仓库筛选）
- **FR-INV-061**: 支持出入库流水查询
- **FR-INV-062**: 支持库存报表导出

### 3.4 销售管理模块

#### 3.4.1 客户管理
- **FR-SAL-001**: 支持客户信息管理（名称、联系人、电话、地址）
- **FR-SAL-002**: 支持客户状态管理（合作中/暂停合作）
- **FR-SAL-003**: 支持客户列表查询与导出

#### 3.4.2 销售订单管理
- **FR-SAL-010**: 支持创建销售订单（选择客户、添加销售明细）
- **FR-SAL-011**: 支持销售订单审核
- **FR-SAL-012**: 支持销售订单状态跟踪（待审核→已审核→部分出库→已完成→已关闭）
- **FR-SAL-013**: 支持销售订单打印

#### 3.4.3 销售出库
- **FR-SAL-020**: 支持根据销售订单生成出库单
- **FR-SAL-021**: 支持部分出库
- **FR-SAL-022**: 出库后自动扣减库存

#### 3.4.4 销售退货
- **FR-SAL-030**: 支持创建销售退货单
- **FR-SAL-031**: 退货后自动增加库存

#### 3.4.5 收款管理
- **FR-SAL-040**: 支持按销售订单登记收款
- **FR-SAL-041**: 支持查看客户欠款情况

#### 3.4.6 销售统计
- **FR-SAL-050**: 支持按时间、客户、商品维度统计销售额
- **FR-SAL-051**: 支持销售排行榜（商品排行、客户排行）
- **FR-SAL-052**: 支持销售明细报表导出

### 3.5 财务管理模块

#### 3.5.1 应收账款
- **FR-FIN-001**: 支持查看客户应收账款汇总
- **FR-FIN-002**: 支持收款登记与核销
- **FR-FIN-003**: 支持应收账款明细查询

#### 3.5.2 应付账款
- **FR-FIN-010**: 支持查看供应商应付账款汇总
- **FR-FIN-011**: 支持付款登记与核销
- **FR-FIN-012**: 支持应付账款明细查询

#### 3.5.3 费用管理
- **FR-FIN-020**: 支持日常费用录入（房租、水电、工资等）
- **FR-FIN-021**: 支持费用分类管理
- **FR-FIN-022**: 支持费用统计报表

#### 3.5.4 经营报表
- **FR-FIN-030**: 支持收支汇总报表（按月/季/年）
- **FR-FIN-031**: 支持利润统计报表
- **FR-FIN-032**: 支持报表导出（Excel）

## 4. 非功能需求

### 4.1 性能需求
- **NFR-001**: 页面加载时间 ≤ 3秒
- **NFR-002**: 普通API响应时间 ≤ 1秒
- **NFR-003**: 报表查询响应时间 ≤ 5秒
- **NFR-004**: 支持50并发用户在线操作

### 4.2 可用性需求
- **NFR-010**: 系统可用性 ≥ 99.5%
- **NFR-011**: 支持数据备份

### 4.3 安全需求
- **NFR-020**: 所有通信采用HTTPS加密
- **NFR-021**: 密码加密存储（BCrypt）
- **NFR-022**: 支持操作日志审计
- **NFR-023**: JWT Token认证，支持过期与刷新

### 4.4 兼容性需求
- **NFR-030**: 支持主流浏览器：Chrome、Firefox、Edge
- **NFR-031**: 支持分辨率：1366×768 及以上

## 5. 核心业务流程

### 5.1 采购流程
```
创建采购订单 → 审核 → 供应商发货 → 采购入库 → 付款登记
```

### 5.2 销售流程
```
创建销售订单 → 审核 → 销售出库 → 收款登记
```

### 5.3 库存管理流程
```
采购入库/退货入库 → 库存更新 → 销售出库/报损出库 → 定期盘点 → 差异调整
```
