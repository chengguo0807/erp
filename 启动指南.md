## 启动步骤

### 1. 完成后端 Maven 编译

在 PowerShell 或 CMD 中执行：

```powershell
cd e:\ERP\enterprisehub-erp
mvn clean install -DskipTests
```

**预计时间**：3-5分钟（首次编译会下载依赖）

---

### 2. 启动 MySQL 和 Redis

#### 方式一：使用 Docker Compose（推荐）

```powershell
cd e:\ERP\enterprisehub-erp
docker-compose up -d mysql redis
```

#### 方式二：手动启动本地服务

- 启动本地 MySQL 服务（端口 3306）
- 启动本地 Redis 服务（端口 6379）

---

### 3. 执行数据库初始化脚本

#### 方式一：使用 MySQL 命令行

```powershell
mysql -u root -p < e:\ERP\enterprisehub-erp\sql\init.sql
```

#### 方式二：使用 MySQL 客户端工具

1. 打开 Navicat / DBeaver / MySQL Workbench
2. 连接到 MySQL（localhost:3306）
3. 打开并执行 `e:\ERP\enterprisehub-erp\sql\init.sql`

**脚本内容**：
- 创建数据库 `erp_system`
- ���建 7 张系统管理表
- 插入初始数据（admin 用户、角色、菜单等）

---

### 4. 修改后端配置（如需要）

编辑 `e:\ERP\enterprisehub-erp\erp-system\src\main\resources\application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/erp_system
    username: root
    password: 123456
  
  redis:
    host: localhost
    port: 6379
    password: 123456
```

---

### 5. 启动后端服务

#### 方式一：使用 Maven 启动（开发模式）

```powershell
cd e:\ERP\enterprisehub-erp\erp-system
mvn spring-boot:run
```

#### 方式二：使用 JAR 包启动

```powershell
cd e:\ERP\enterprisehub-erp
java -jar erp-system/target/erp-system.jar
```

**启动成功标志**：
- 看到 "Started SystemApplication in X seconds"
- 服务运行在 http://localhost:8082

---

### 6. 启动前端开发服务器

打开新的 PowerShell 窗口：

```powershell
cd e:\ERP\enterprisehub-erp-ui
npm run dev
```

**启动成功标志**：
- 看到 "Local: http://localhost:3000/"
- 浏览器自动打开或手动访问 http://localhost:3000

## 常见问题

### Q1: Maven 编译失败
- 检查 JDK 版本是否为 17+：`java -version`
- 检查 Maven 版本：`mvn -version`
- 清理缓存：`mvn clean`

### Q2: MySQL 连接失败
- 检查 MySQL 是否启动：`mysql -u root -p`
- 检查端口是否被占用：`netstat -ano | findstr 3306`
- 检查用户名密码是否正确

### Q3: Redis 连接失败
- 检查 Redis 是否启动：`redis-cli ping`
- 检查端口：`netstat -ano | findstr 6379`

### Q4: 前端页面空白
- 检查后端服务是否启动
- 打开浏览器控制台查看错误信息
- 检查 API 请求是否正常

### Q5: 端口被占用
- 查看端口占用：`netstat -ano | findstr 8082`
- 杀掉进程：`taskkill /PID <进程ID> /F`
